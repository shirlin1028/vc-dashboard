# Sales Dashboard (VibeAnalytics)

A modern, clean SaaS-style dashboard built with React and Tailwind CSS. The backend API is configured to read data from a **local CSV by default**, but is fully ready to connect to **Supabase**.

## 🚀 Features

- **Intelligence Center**: AI-powered insights using Google Gemini 1.5/3.1 models.
- **Pulse Insights**: Real-time identification of best products, channels, and peak days.
- **Hybrid Data Layer**: Seamlessly switch between local CSV storage or Supabase Cloud DB.
- **Modern Aesthetics**: Sleek dark mode interface with rich charts and high-quality icons.

##  Installation & Setup

1. **Clone the repository** (if applicable) or navigate to the project folder.
2. **Install dependencies**:
   ```bash
   npm install
   ```
3. **Configure Gemini AI**:
   Add your Gemini API key to your `.env` file:
   ```env
   VITE_GEMINI_API_KEY=your_gemini_key
   ```
4. **Run the development server**:
   ```bash
   npm run dev
   ```
5. **Access the dashboard**: Open [http://localhost:5175/](http://localhost:5175/) in your browser.

## 🗄️ How to Connect Supabase (For Non-Coders)

The application is cloud-ready! Follow these simple steps to move from a static CSV to a live database:

1. **Create your project**: Go to [Supabase](https://supabase.com/) and create a new project.
2. **Setup the table**: In the **SQL Editor**, create the `sales_data` table by copy-pasting and running this snippet:
   ```sql
   CREATE TABLE sales_data (
       id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
       date DATE NOT NULL,
       product TEXT NOT NULL,
       channel TEXT NOT NULL,
       orders INTEGER DEFAULT 0,
       revenue DECIMAL(12, 2) DEFAULT 0.00,
       cost DECIMAL(12, 2) DEFAULT 0.00,
       visitors INTEGER DEFAULT 0,
       customers INTEGER DEFAULT 0,
       created_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL
   );
   ```
3. **Import your data**: Open the **Table Editor**, select your new `sales_data` table, and use the **Import CSV** tool to upload your `sales_data.csv`.
4. **Get your credentials**: Go to **Project Settings -> API** to get your **Project URL** and **Anon Key**.
5. **Update your settings**: Open your local `.env` file and update your credentials:
   ```env
   DATA_SOURCE=supabase
   SUPABASE_URL=your_project_url
   SUPABASE_ANON_KEY=your_anon_key
   ```
6. **Restart & Enjoy**: Restart your development server (`npm run dev`). The API will now serve data directly from Supabase!
